# Simon Memory Game STM32

A simple embedded Simon Says memory game built with STM32CubeIDE for the
`STM32F091RC` Nucleo board.

The game plays an LED sequence across four colored positions, then waits for the
player to repeat that sequence using four push buttons. Each successful round
adds one more step to the pattern. A wrong input triggers a game-over flash
animation and reports the score over UART.

## Hardware

- Board: `NUCLEO-F091RC`
- IDE/toolchain: `STM32CubeIDE`
- Programmer/debugger: onboard ST-LINK
- Other: Breadboard, 4 push buttons, jumper wires, x4 100 Ohm resistors, 4 LED's

## Gameplay

1. Press the onboard user button on `PC13` to start the game.
2. The board flashes the current Simon sequence across the four LEDs.
3. Repeat the sequence using the four external player buttons.
4. Each correct round increases the sequence length by one.
5. A wrong input triggers the fail animation, prints score details over UART,
  and waits for the user button to restart.

## Pin Mapping


| Function               | STM32 Pin | Notes                            |
| ---------------------- | --------- | -------------------------------- |
| LED 1                  | `PA0`     | GPIO output                      |
| LED 2                  | `PA1`     | GPIO output                      |
| LED 3                  | `PA4`     | GPIO output                      |
| LED 4                  | `PA5`     | GPIO output, can use onboard LD2 |
| Player button 1        | `PB0`     | GPIO input, pull-up, active low  |
| Player button 2        | `PB1`     | GPIO input, pull-up, active low  |
| Player button 3        | `PB4`     | GPIO input, pull-up, active low  |
| Player button 4        | `PB5`     | GPIO input, pull-up, active low  |
| Start / restart button | `PC13`    | Nucleo user button, pull-up      |
| UART2 TX               | `PA2`     | Serial debug output              |
| UART2 RX               | `PA3`     | Serial input line                |


Connect each external LED through a resistor. The button inputs are configured as pull-up inputs, so each button press is read as active low.

## Project Structure

- `SimonGame.ioc` - STM32CubeMX / CubeIDE configuration for the board, clocks,
GPIO, and UART.
- `Core/Src/main.c` - Simon game logic, sequence generation, input handling,
LED control, and UART status messages.
- `Core/Inc/main.h` - generated HAL includes and board pin definitions.
- `Drivers/` - STM32 HAL and CMSIS files generated by STM32CubeIDE.
- `SimonGame Debug.launch` - CubeIDE debug launch configuration for ST-LINK.
- `STM32F091RCTX_FLASH.ld` - linker script for the target MCU.



## Building and Flashing

1. Install `STM32CubeIDE`.
2. Clone this repository.
3. Open STM32CubeIDE and choose `File > Import`.
4. Select `Existing Projects into Workspace`.
5. Browse to the cloned repository folder and import the `SimonGame` project.
6. Build the project with `Project > Build Project`.
7. Connect the `NUCLEO-F091RC` board over USB.
8. Flash or debug the firmware with `Run > Run` or `Run > Debug`.



## Configuration Notes

- The MCU is configured as `STM32F091RCTx`.
- The system clock is configured for `48 MHz` using the internal HSI with PLL.
- UART debug output uses `USART2` at `115200` baud on `PA2` / `PA3`.
- The game stores up to `100` sequence entries in memory.
- The LED sequence and player input are handled with HAL GPIO APIs in the main
loop without an RTOS.
